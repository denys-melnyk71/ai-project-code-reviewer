# AI Code Reviewer (Senior Developer Role)

Автоматизований інструмент для проведення комплексного аналізу та рев'ю вихідного Python-коду. Застосунок використовує мовну модель **Gemini 2.5 Flash** через офіційний SDK `google-genai` для імітації ролі досвідченого **Senior Python Developer / Tech Lead**.

Веб-інтерфейс побудований на базі фреймворку **Streamlit** з інтеграцією локального кешування, що мінімізує навантаження на ліміти безкоштовного тарифу (Free Tier).

---

## Основні можливості

* **Комплексне рев'ю за 3 критеріями:**
1. **Відповідність стандартам PEP 8** — аналіз стилістики, правильних відступів, іменування змінних, функцій та класів.
2. **Безпека та вразливості** — пошук критичних проблем у коді (SQL-ін'єкції, жорстко закодовані паролі/токени, використання небезпечних функцій типу `eval`, `exec`).
3. **Рефакторинг та Clean Code** — пошук можливостей оптимізації алгоритмів, дотримання принципів DRY (Don't Repeat Yourself) та покращення читаємості.


* **Гнучкі методи введення коду:** завантаження готового файлу з розширенням `.py` або ручне копіювання/введення фрагментів у текстове поле.
* **Локальне кешування запитів:** інтегрований механізм `st.cache_data` запобігає повторним зверненням до API, якщо аналізований код не змінювався. Можливість примусового очищення кешу через бічну панель (Sidebar).
* **Суворий структурований вивід (Structured Output):** відповіді моделі повністю валідуються на базі схем даних **Pydantic**, що гарантує 100% стабільність інтерфейсу без ризику збоїв чи галюцинацій текстового формату ШІ.
* **Локалізація:** Усі аналітичні звіти, зауваження та коментарі ментора генеруються **українською мовою**.

## Системні вимоги та встановлення

Застосунок розроблено на базі сучасного інструментарію Python. Для ізоляції залежностей, швидкого пакування та гарантії відтворюваності середовища на будь-якій машині використовується менеджер пакетів **`uv`** від Astral.

### Попередні вимоги
Перед початком переконайтеся, що у вашій системі встановлено:
* **Python** версії `3.10` або новішої.
* **Git** (для клонування репозиторію).

---

### Крок 1: Встановлення менеджера пакетів `uv`

`uv` — це надшвидка альтернатива традиційним `pip` та `venv`. Вона автоматично керує версіями Python та віртуальними середовищами. Інсталюйте `uv` залежно від вашої операційної системи:

* **Windows (PowerShell):**
  ```powershell
  powershell -ExecutionPolicy ByPass -c "irm https://astral.sh/uv/install.ps1 | iex"
  ```

* **Linux / macOS:**
  ```bash
  curl -LsSf https://astral.sh/uv/install.sh | sh
  ```



> *Після встановлення рекомендується перезапустити термінал, щоб оновилися змінні оточення (PATH).*

---

### Крок 2: Клонування репозиторію

Завантажте вихідний код проєкту на свій локальний комп'ютер за допомогою Git:

```bash
git clone https://github.com/denys-melnyk71/ai-project-code-reviewer
cd ai-project-code-reviewer
```

---

### Крок 3: Створення середовища та встановлення залежностей

Завдяки конфігураційному файлу `pyproject.toml`, вам **не потрібно** вручну створювати віртуальне оточення чи по черзі встановлювати бібліотеки. `uv` зробить усе автоматично.

Просто виконайте команду синхронізації:

```bash
uv sync
```

## Налаштування та запуск застосунку

Перед запуском необхідно отримати безкоштовний ключ доступу **Gemini API** в особистому кабінеті [Google AI Studio](https://aistudio.google.com/).

### Крок 1. Налаштування змінної оточення (API-ключа)

Встановіть отриманий ключ у терміналі вашої операційної системи:

* **Для Windows (PowerShell):**
```powershell
$env:GEMINI_API_KEY="ваш_реальний_api_ключ_від_google"
```

* **Для Windows (CMD):**
```cmd
set GEMINI_API_KEY=ваш_реальний_api_ключ_від_google
```

* **Для Linux / macOS (Bash/Zsh):**
```bash
export GEMINI_API_KEY="ваш_реальний_api_ключ_від_google"
```

### Крок 2. Запуск через `uv run`

Запустіть сервер Streamlit за допомогою наступної команди:

```bash
uv run streamlit run src/reviewer/main.py
```

Після виконання команди `uv` автоматично підготує ізольоване середовище і відкриє застосунок у вашому браузері за замовчуванням за адресою: **`http://localhost:8501`**.

## Обробка помилок (Error Handling)

Застосунок стійкий до збоїв та передбачає детальну обробку помилок з виведенням зрозумілих повідомлень користувачу:

1. **Відсутність токена/Невалідний токен (Коди 400/403):** Програма аналізує текстове повідомлення від `APIError` та чітко вказує на проблему з `GEMINI_API_KEY`.
2. **Перевищення лімітів (Код 429 - Resource Exhausted):** Система інформує про вичерпання квот Free Tier та просить зачекати перед наступним запитом.
3. **Відсутність інтернету / Мережевий збій:** Завдяки перехопленню винятків `httpx.ConnectError` та `httpx.NetworkError`, у разі обриву зв'язку користувач побачить лаконічне сповіщення про перевірку підключення до мережі замість критичного аварійного завершення роботи бекенду.
